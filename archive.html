05 · Applications

Website Interface
Database UI
Database-style list of objects. Category codes + numbers as the primary navigation layer. Information structured as datasets, not editorial pages.
B
Classification
Badges & Icons
Category pills, 3D cube icons with abbreviations, color-coded tags. Applied across print and digital touchpoints as the core visual unit.
C
Exhibition

Physical labeling system. Wayfinding on arches and walls. The identity guides movement and interpretation in space.

Merchandise

Tote bags, notebooks, ID cards — each carrying category codes and numbers. The system reproduces as physical objects.
06 · The Space — Playground & Classroom
Three modes
of space
The three modes of space work on an escalating logic: from physical database to workshop environment to projected architecture.
01
Archive Mode
Shelving units with color-coded classification as a physical database. The body moves through the system in the same way that a cursor moves through a website.
02
Workshop Mode
Numbered stations 001 / 002 / 003 with tablets and materials. Each workstation is an indexed unit of the catalog. A learning environment that is a catalog itself.
03
Stage Mode
A projector suspended above the floor projects building layouts directly onto the surface. Participants physically occupy projected contours — sitting inside buildings that do not yet exist.
Architect→Art Director

This project didn't ask me to stop thinking like an architect — it asked me to apply that thinking to a different material. Classification systems, spatial grids, inventory logic, wayfinding: these are architectural problems.

Translating them into a visual identity system felt like a natural extension rather than a departure. The categories, color logic, and numbering system were designed in Figma. Exhibition and web visualizations were generated with AI to communicate spatial application — the system itself came first.
